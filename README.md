AI models are powerful but can sometimes produce outputs that don't conform to a desired structure, leading to downstream processing errors. This project provides a robust solution: a Structured Output Validator with Auto-Retry. It leverages Pydantic for defining expected data schemas and implements an intelligent retry mechanism to ensure generated AI responses adhere strictly to those schemas.

The core idea is to define your desired data structure using a Pydantic model. When interacting with a Large Language Model (LLM), the system attempts to parse and validate the LLM's output against this model. If validation fails, perhaps due to malformed JSON or incorrect data types, the system automatically re-prompts the LLM, incorporating a brief delay (with exponential backoff) for resilience. This process repeats for a configurable number of attempts, significantly improving the reliability of AI-generated structured data.

To use, set your `GEMINI_API_KEY` environment variable. Then, simply run `main.py`. The demo showcases validating a product information JSON against a Pydantic model, demonstrating how the system retries until a valid output is received. This pattern is essential for building reliable AI-powered applications that depend on structured data.
